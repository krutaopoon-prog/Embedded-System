เครื่องมือจำลองการทำงาน SPI Protocol สำหรับนักเรียน ปวส.

📖 ทฤษฎีก่อนใช้ Simulator

🔌 SPI คืออะไร?

SPI (Serial Peripheral Interface) เป็นโปรโตคอลสื่อสารแบบ Synchronous Serial ที่ใช้สำหรับเชื่อมต่ออุปกรณ์ระยะใกล้ในระบบ Embedded System

📘 คำศัพท์ที่ต้องรู้

  • Synchronous = สื่อสารแบบมีสัญญาณนาฬิกาควบคุมจังหวะ (ต้องมี SCLK)
  • Serial = ส่งข้อมูลทีละบิต ต่อเนื่องกันเป็นสาย (ไม่ใช่แบบ Parallel ที่ส่งพร้อมกันหลายบิต)
  • Full-Duplex = ส่งและรับข้อมูลพร้อมกันได้ในเวลาเดียวกัน (เหมือนโทรศัพท์)

ลักษณะเด่นของ SPI:

  • พัฒนาโดย Motorola ในปี 1970s
  • สื่อสารแบบ Full-Duplex (ส่งและรับพร้อมกัน)
  • ใช้สายสัญญาณ 4 เส้น (แบบ 4-wire)
  • รองรับความเร็วสูง (หลาย MHz)

🎮 Master และ 📟 Slave คืออะไร?

SPI ใช้โครงสร้างแบบ Master-Slave คือมีอุปกรณ์หนึ่งควบคุมการสื่อสาร (Master) และอุปกรณ์อื่นรอรับคำสั่ง (Slave)

📘 ความหมายของ Master และ Slave

🎮 Master (เจ้านาย)

  • อุปกรณ์ที่ควบคุมการสื่อสารทั้งหมด
  • เป็นฝ่ายสร้างสัญญาณนาฬิกา (SCLK)
  • ตัดสินใจเมื่อไหร่จะส่ง/รับข้อมูล
  • เลือกว่าจะคุยกับ Slave ตัวไหน (ผ่านสัญญาณ SS)
  • ตัวอย่าง: Arduino, Raspberry Pi, STM32

📟 Slave (ผู้รับใช้)

  • อุปกรณ์ที่รอรับคำสั่งจาก Master
  • ไม่มีสิทธิ์สร้างสัญญาณนาฬิกา
  • ตอบสนองต่อ Master เท่านั้นเมื่อถูกเลือก (SS = LOW)
  • ตัวอย่าง: Sensor (เซนเซอร์), EEPROM, Display

📡 สายสัญญาณ 4 เส้น คืออะไร?

SPI ใช้สายสัญญาณ 4 เส้น แต่ละเส้นมีชื่อและหน้าที่เฉพาะเจาะจง

🟢 MOSI - Master Out Slave In

  • Master Out Slave In = ข้อมูลออกจาก Master เข้าไปใน Slave
  • ทิศทาง: จาก Master → ไปยัง Slave (ขาเดียว)
  • หน้าที่: Master ใช้สายนี้ ส่งข้อมูล ไปให้ Slave
  • คิดง่ายๆ: สายส่งข้อมูลจากเจ้านายไปลูกน้อง

🔵 MISO - Master In Slave Out

  • Master In Slave Out = ข้อมูลเข้า Master ออกจาก Slave
  • ทิศทาง: จาก Slave → กลับไป Master (ขาเดียว)
  • หน้าที่: Slave ใช้สายนี้ ส่งข้อมูลกลับ ไปให้ Master
  • คิดง่ายๆ: สายส่งข้อมูลจากลูกน้องกลับไปหาเจ้านาย

🟣 SCLK - Serial Clock

  • Serial Clock = สัญญาณนาฬิกาสำหรับซิงโครไนซ์
  • ทิศทาง: จาก Master → ไปยัง Slave
  • หน้าที่: เป็นจังหวะจะโคนบอก Slave ว่าเมื่อไหร่ควรอ่านข้อมูล
  • คิดง่ายๆ: คือจังหวะเพลงที่ Master นำขึ้นมาบรรเลง ทุกคนต้องตามจังหวะนี้
  • ความถี่: ยิ่งเร็ว = ส่งข้อมูลเร็ว (วัดเป็นหน่วย Hz เช่น 1 MHz, 8 MHz)

🔴 SS (CS) - Slave Select / Chip Select

  • Slave Select หรือ Chip Select = เลือก Slave ที่จะคุยด้วย
  • ทิศทาง: จาก Master → ไปยัง Slave
  • หน้าที่: Master ใช้เลือกว่าจะคุยกับ Slave ตัวไหน
  • Active LOW: เมื่อ SS = LOW (0V) = เลือก Slave นี้, SS = HIGH (3.3V/5V) = ไม่เลือก
  • คิดง่ายๆ: เหมือนการตะโกนชื่อคน ถ้าเรียกชื่อคนไหน คนนั้นถึงจะตอบ

⏱️ Rising Edge และ Falling Edge คืออะไร?

สัญญาณนาฬิกา (SCLK) เป็นคลื่นสี่เหลี่ยมที่สลับระหว่าง HIGH และ LOW การเปลี่ยนระดับนี้เรียกว่า "Edge"

📘 คำอธิบาย Edge

⬆️ Rising Edge (ขาขึ้น)

  • สัญญาณเปลี่ยนจาก LOW → HIGH (0 → 1)
  • เหมือนขึ้นบันไดจากพื้นขึ้นไปชั้นบน
  • ใน SPI Mode 0 และ 3: อ่านข้อมูลที่ Rising Edge

⬇️ Falling Edge (ขาลง)

  • สัญญาณเปลี่ยนจาก HIGH → LOW (1 → 0)
  • เหมือนเดินลงบันไดจากชั้นบนลงมาพื้น
  • ใน SPI Mode 1 และ 2: อ่านข้อมูลที่ Falling Edge

💡 ทำไมต้องมี Edge? การอ่านข้อมูลที่ Edge ช่วยให้อ่านตรงจังหวะที่สัญญาณคงที่ ไม่ใช่ตอนที่กำลังเปลี่ยน (ซึ่งอาจอ่านค่าผิด)

⚙️ SPI Modes (CPOL & CPHA) คืออะไร?

SPI มี 4 Modes ตามการตั้งค่าสองตัว: CPOL (Clock Polarity) และ CPHA (Clock Phase)

📘 ความหมาย CPOL และ CPHA

CPOL - Clock Polarity (ความถูกต้องของนาฬิกา)

  • กำหนดว่าสัญญาณนาฬิกาเริ่มต้นที่ระดับไหน
  • CPOL = 0: เริ่มที่ LOW (0V) เมื่อไม่ส่งข้อมูล
  • CPOL = 1: เริ่มที่ HIGH (3.3V/5V) เมื่อไม่ส่งข้อมูล

CPHA - Clock Phase (เฟสของนาฬิกา)

  • กำหนดว่าอ่านข้อมูลที่ Edge ไหน
  • CPHA = 0: อ่านที่ Rising Edge, เปลี่ยนข้อมูลที่ Falling Edge
  • CPHA = 1: อ่านที่ Falling Edge, เปลี่ยนข้อมูลที่ Rising Edge
Mode CPOL CPHA เริ่มต้น อ่านข้อมูลที่
0 0 0 LOW Rising Edge
1 0 1 LOW Falling Edge
2 1 0 HIGH Falling Edge
3 1 1 HIGH Rising Edge

💡 Simulator นี้รองรับ Mode 0 และ 3 เพราะทั้งสอง Mode อ่านข้อมูลที่ Rising Edge (ใช้งานง่ายที่สุด)

🔄 ขั้นตอนการส่งข้อมูล (Data Transfer)

การส่งข้อมูล 1 byte (8 bits) ใน SPI ทำตามขั้นตอนนี้:

  1. Master ส่ง SS = LOW (เลือก Slave)

    บอก Slave ว่า "เดี๋ยวจะคุยกับนายนะ" - Slave ที่ถูกเลือกจะเริ่มตั้งตัวรอ

  2. เริ่มส่งสัญญาณนาฬิกา (SCLK)

    Master เริ่มสร้างคลื่นสี่เหลี่ยมที่ SCLK เป็นลูกๆ ต่อเนื่อง

  3. ส่งข้อมูลบิตต่อบิต (MSB-first)

    เริ่มจากบิตที่มีค่ามากที่สุด (Most Significant Bit) ไปหาน้อยที่สุด

    ตัวอย่าง: ส่ง 0xAA (10101010) จะส่ง 1→0→1→0→1→0→1→0

  4. ทุก clock cycle: ส่งและรับพร้อมกัน

    ที่ Rising Edge: Master อ่าน MISO + Slave อ่าน MOSI พร้อมกัน

    นี่แหละคือ Full-Duplex - ส่งและรับพร้อมกัน!

  5. ครบ 8 cycles = จบ 1 byte

    ส่งข้อมูลครบ 8 บิตแล้ว หยุดสัญญาณนาฬิกา

  6. Master ส่ง SS = HIGH (ปล่อย Slave)บอก Slave ว่า "คุยเสร็จแล้ว นายไปได้" - จบการสื่อสาร

⚡ สรุป: 1 byte = 8 bits = 8 clock cycles = ส่งและรับพร้อมกัน 8 ครั้ง

💻 ตัวอย่างโค้ด Arduino
#include <SPI.h>

void setup() {
  SPI.begin();
  pinMode(SS, OUTPUT);
  digitalWrite(SS, HIGH);
}

void loop() {
  digitalWrite(SS, LOW);  // เลือก Slave
  SPI.transfer(0xAA);     // ส่งข้อมูล 0xAA
  digitalWrite(SS, HIGH); // ปล่อย Slave
  delay(1000);
}
#include <SPI.h>

volatile byte received;

void setup() {
  pinMode(MISO, OUTPUT);
  SPCR |= _BV(SPE);
  SPI.attachInterrupt();
  Serial.begin(9600);
}

ISR(SPI_STC_vect) {
  received = SPDR;
  Serial.println(received, HEX);
}

void loop() {
  // Slave ทำงานอัตโนมัติผ่าน interrupt
}

คำถามที่ 1: สายสัญญาณใดเป็นสายข้อมูลจาก Master ไป Slave?

✅ MOSI ย่อมาจาก Master Out Slave In - เป็นสายข้อมูลที่ Master ส่งไปยัง Slave

คำถามที่ 2: สายสัญญาณใดใช้สำหรับเลือกอุปกรณ์ Slave ที่จะสื่อสาร?

✅ SS (Slave Select) หรือ CS (Chip Select) ใช้สำหรับเลือก Slave ที่ Master ต้องการสื่อสารด้วย (Active LOW)

คำถามที่ 3: การสื่อสาร SPI เป็นแบบใด?

✅ SPI เป็น Full-Duplex - สามารถส่งข้อมูลผ่าน MOSI และรับข้อมูลผ่าน MISO พร้อมกันในแต่ละ clock cycle

คำถามที่ 7: สัญญาณ SS ต้องเป็นอย่างไรก่อนเริ่มส่งข้อมูล?

✅ SS เป็น Active LOW - ต้องเป็น LOW เพื่อเลือก Slave ที่ต้องการสื่อสาร

คำถามที่ 8: ถ้า SCLK ความถี่ 1 MHz จะใช้เวลาส่ง 1 byte นานเท่าใด?

✅ 1 byte = 8 bits, 1 MHz = 1 bit/ไมโครวินาที ดังนั้น 8 bits = 8 ไมโครวินาที

คำถามที่ 9: ถ้ามีหลาย Slave ต้องใช้สายสัญญาณเพิ่มอะไร?

✅ ต้องมี SS/CS แยกสำหรับแต่ละ Slave แต่ MOSI, MISO, SCLK ใช้ร่วมกันได้

คำถามที่ 10: ข้อใดเป็นข้อดีของ SPI เมื่อเทียบกับ I2C?

✅ SPI เร็วกว่าและเป็น Full-Duplex แต่ใช้สายมากกว่า I2C
