🔌 SPI คืออะไร?

SPI (Serial Peripheral Interface) เป็นโปรโตคอลสื่อสารแบบ Synchronous Serial ที่ใช้สำหรับเชื่อมต่ออุปกรณ์ระยะใกล้ในระบบ Embedded System

  • พัฒนาโดย Motorola ในปี 1970s
  • สื่อสารแบบ Full-Duplex (ส่งและรับพร้อมกัน)
  • ใช้สายสัญญาณ 4 เส้น (แบบ 4-wire)
  • รองรับความเร็วสูง (หลาย MHz)

📡 สายสัญญาณ 4 เส้น

สัญญาณ ทิศทาง หน้าที่
MOSI Master → Slave ส่งข้อมูลจาก Master ไป Slave
MISO Slave → Master ส่งข้อมูลจาก Slave กลับ Master
SCLK Master → Slave สัญญาณนาฬิกาสำหรับซิงค์
SS (CS) Master → Slave เลือก Slave ที่จะสื่อสาร (Active LOW)

⚙️ SPI Modes (CPOL & CPHA)

SPI มี 4 Modes ตามการตั้งค่า Clock Polarity (CPOL) และ Clock Phase (CPHA):

Mode CPOL CPHA คำอธิบาย
0 0 0 เริ่ม LOW, อ่านที่ rising edge
1 0 1 เริ่ม LOW, อ่านที่ falling edge
2 1 0 เริ่ม HIGH, อ่านที่ falling edge
3 1 1 เริ่ม HIGH, อ่านที่ rising edge

💡 Simulator นี้รองรับ Mode 0 และ 3 (ทั้งคู่ sample ที่ rising edge)

🔄 การส่งข้อมูล (Data Transfer)

  1. Master ส่งสัญญาณ SS = LOW เพื่อเลือก Slave
  2. ส่งข้อมูลบิตต่อบิต แบบ MSB-first (บิตสูงสุดก่อน)
  3. ทุก clock cycle ส่งและรับพร้อมกัน 1 บิต
  4. ครบ 8 cycles = ส่งข้อมูล 1 byte เสร็จสมบูรณ์
  5. Master ส่ง SS = HIGH เพื่อจบการสื่อสาร

⚡ 1 byte = 8 bits = 8 clock cycles

💻 ตัวอย่างโค้ด Arduino
#include <SPI.h>

void setup() {
  SPI.begin();
  pinMode(SS, OUTPUT);
  digitalWrite(SS, HIGH);
}

void loop() {
  digitalWrite(SS, LOW);  // เลือก Slave
  SPI.transfer(0xAA);     // ส่งข้อมูล 0xAA
  digitalWrite(SS, HIGH); // ปล่อย Slave
  delay(1000);
}
#include <SPI.h>

volatile byte received;

void setup() {
  pinMode(MISO, OUTPUT);
  SPCR |= _BV(SPE);
  SPI.attachInterrupt();
  Serial.begin(9600);
}

ISR(SPI_STC_vect) {
  received = SPDR;
  Serial.println(received, HEX);
}

void loop() {
  // Slave ทำงานอัตโนมัติผ่าน interrupt
}

คำถามที่ 1: สายสัญญาณใดเป็นสายข้อมูลจาก Master ไป Slave?

✅ MOSI ย่อมาจาก Master Out Slave In - เป็นสายข้อมูลที่ Master ส่งไปยัง Slave

คำถามที่ 2: สายสัญญาณใดใช้สำหรับเลือกอุปกรณ์ Slave ที่จะสื่อสาร?

✅ SS (Slave Select) หรือ CS (Chip Select) ใช้สำหรับเลือก Slave ที่ Master ต้องการสื่อสารด้วย (Active LOW)

คำถามที่ 3: การสื่อสาร SPI เป็นแบบใด?

✅ SPI เป็น Full-Duplex - สามารถส่งข้อมูลผ่าน MOSI และรับข้อมูลผ่าน MISO พร้อมกันในแต่ละ clock cycle

✅ SCLK = นาฬิกา, MOSI = Master ส่งให้ Slave, MISO = Master รับจาก Slave

คำถามที่ 5: ถ้าต้องการให้ SCLK เริ่มต้นที่ LOW และอ่านข้อมูลที่ rising edge ควรใช้โหมดใด?

✅ Mode 0: CPOL=0 (SCLK เริ่ม LOW), CPHA=0 (อ่านที่ rising edge)

คำถามที่ 6: จาก waveform นี้ ข้อมูลที่ส่งคือค่าใด? (LSB first)

0 1 2 3 4 5 6 7
✅ ข้อมูลที่ rising edge: บิต 0=0, 1=0, 2=1, 3=1, 4=0, 5=0, 6=1, 7=1 = 00110011 (0x33)

คำถามที่ 7: สัญญาณ SS ต้องเป็นอย่างไรก่อนเริ่มส่งข้อมูล?

✅ SS เป็น Active LOW - ต้องเป็น LOW เพื่อเลือก Slave ที่ต้องการสื่อสาร

คำถามที่ 8: ถ้า SCLK ความถี่ 1 MHz จะใช้เวลาส่ง 1 byte นานเท่าใด?

✅ 1 byte = 8 bits, 1 MHz = 1 bit/ไมโครวินาที ดังนั้น 8 bits = 8 ไมโครวินาที

คำถามที่ 9: ถ้ามีหลาย Slave ต้องใช้สายสัญญาณเพิ่มอะไร?

✅ ต้องมี SS/CS แยกสำหรับแต่ละ Slave แต่ MOSI, MISO, SCLK ใช้ร่วมกันได้

คำถามที่ 10: ข้อใดเป็นข้อดีของ SPI เมื่อเทียบกับ I2C?

✅ SPI เร็วกว่าและเป็น Full-Duplex แต่ใช้สายมากกว่า I2C
